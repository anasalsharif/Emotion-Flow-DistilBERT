Emotion Flow: Real-time Emotion Analyzer

Final Project Report - Building an Intelligent Application

1. Problem Statement

In this project, I developed Emotion Flow, a real-time emotion classification dashboard. The goal was to build a modern, scalable NLP application that can accurately detect complex emotional nuances in text (joy, sadness, anger, fear, love, surprise). This aligns with Track 1: Natural Language Processing from the assignment guidelines.

2. Dataset and Preprocessing

We utilized the dair-ai/emotion dataset from HuggingFace, a highly regarded dataset consisting of English Twitter messages mapped to 6 basic emotions.

The NLP Preprocessing Pipeline

To demonstrate mastery of both classic and modern NLP, I implemented a custom multi-stage preprocessing pipeline:

  • Normalization (Regex): We used Regular Expressions to strip exaggerated punctuation (e.g., !!! to !) and reduce repeated character strings (e.g., soooo to so). This reduces noise and improves model consistency.
  • Spell Correction (Edit Distance): We implemented the Levenshtein Distance algorithm. For each word not found in our curated vocabulary, the system calculates the edit distance to identify and correct common typos (e.g., hapy to happy) before the text reaches the model.
  • Tokenization & Embeddings: Text is converted into numerical IDs using the AutoTokenizer, which are then mapped into high-dimensional embeddings that represent the semantic meaning of each word.

3. Model Architecture

The core system leverages DistilBERT (distilbert-base-uncased), a streamlined version of the original BERT model.

  • Why DistilBERT? It reduces the model size by 40% while retaining 97% of its language understanding capabilities and being 60% faster. This demonstrates efficiency in deployment, a key part of modern AI.
  • Architecture Details: It uses standard Transformer blocks (Multi-Head Self-Attention mechanisms) but with fewer layers. The final hidden state of the [CLS] token is passed through a Linear layer (Classification Head) mapping to our 6 emotion classes.

4. The Training Process

The model was fine-tuned using the PyTorch framework and HuggingFace's Trainer API.

  • Loss Function: We used Cross-Entropy Loss, standard for multi-class classification tasks.
  • Optimizer & Gradient Descent: The model was trained using the AdamW optimizer, a sophisticated variant of Gradient Descent with weight decay.
  • Backpropagation: During each training mini-batch, the forward pass computes predictions. The loss is then calculated against the true labels. Backpropagation is used to compute the gradients of the loss with respect to all the model's weights. The optimizer then updates the weights in the negative direction of the gradient to minimize the loss over epochs.

5. Evaluation Metrics

The model was evaluated on a held-out test set from the dair-ai/emotion corpus.

  • Classification Accuracy: 92.7%
  • F1 Score: 92.6% (weighted average)
  • Performance: Inference latency is <150ms on a standard CPU, proving the efficiency of the DistilBERT architecture for production environments.
